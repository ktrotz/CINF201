Introduction to Arrays

An array is a way to have a single variable store multiple pieces of information with a sequential index. It is like having a variable that behaves as a list. Keep in mind that the index enumerations with start with 0.

Note: this rainbow array begins imperfectly in order to demonstrate throughout the webpage how to manipulate the array. 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'purple', 'pink' ];

alert( rainbow[0] );

The square-brackets – [ and ] – are used in two different ways here. First, in defining the whole list, they are the delimiters. That is, they start and end the whole list. Then, later, when retrieving the information, they are attached to the name of the variable and delimit the index (or position) on that list. One sees 'red' because 'red' is in position 0 in the rainbow array, just as 'orange' is in position 1 in the rainbow array.

How many elements are in this array?

The number of elements in an array can be accessed using the .length field inherent to every array. The array below has 7 elements. 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'purple', 'pink' ];

alert( 'rainbow has ' + rainbow.length + ' elements.');

The .length field can be a useful property, as will be seen in upcoming sections. Also, one must be careful in distinguishing the .length value with the indexing of elements within the array itself. 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'purple', 'pink' ];

alert( 'rainbow[7] is ' + rainbow[7]);

This undefined response is to be expected. If counting starts at 0, then it will naturally end one less than the .length value.

Finding an Index of an Array Element

First, let's take on the issue that it should be 'violet', not 'purple'. So, we would need to identify the index at which the erroneous value ('purple') exists, and then (re)assign the current value ('violet') at that index.

Locating an Array Element 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'purple', 'pink' ];

alert( rainbow.indexOf('purple') );

This should yield the index 5 (because it is currently the 6th element in the array). Let's modify the code, using this information, to assign the correct value. 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'purple', 'pink' ];

alert( 'before:\n' + rainbow );

rainbow[ rainbow.indexOf('purple') ] = 'violet';

alert( 'after:\n' + rainbow );

The .indexOf() array method, applied to the rainbow array, looks for the value given ('purple'), and returns the index (integer) 5. This is then passed into the sqaure-brackets to locate that part of the array (list of values). The reassignment of the values is then completed.

Note: The .indexOf() array method will return only the first index at which a match is made. There also exists an analogous .lastIndexOf() array method too. To locate additional indices at which a match is made requires more involved coding.

Inserting or Deleting Array Elements

Let's address the missing 'green' value. But first, we need to learn about a number of array methods.

Useful Array Methods for Editing
.indexOf( value ) returns the index of the first match for the value given.
.lastIndexOf( value ) returns the index of the first match for the value given,
counting from the end.
.push( value ) extends the array by one element,
pushing the value into the final index.
.pop( ) reduces the array by one element,

.unshift( value ) extends the array by one element,
pushing the value into the first index.
.shift( ) reduces the array by one element,
removing the value from the first index.
.splice( special ) .splice( I, N, item1, ..., itemX )
the I determines the first index to eliminate elements from.
the N is the number of elements, starting at I, to eliminate.
then index inserts item1 at index I
then item2 is inserted at index I+1
then item3 is inserted at index I+2
and so on until
then itemX is inserted at index I+X–1
and the element formerly at index I+N is now at I+X
and the element formerly at index I+N+1 is now at I+X+1
and so on.

One would think that for a single new element a simple .insert( ) would suffice, but alas, one must use .splice(index,0,newValue). 

rainbow = [ 'red', 'orange', 'yellow', 'blue', 
           'indigo', 'violet', 'pink' ];

alert( 'before:\n' + rainbow 
            + '\nlength: ' + rainbow.length );

rainbow.splice(3,0,'green');

alert( 'after:\n' + rainbow 
           + '\nlength: ' + rainbow.length );

Now, to delete an array element (in our case, the 'pink'). While we could exploit the fact that it is the last element of the array and just .pop( ) it away, there are two common methods for doing this in a more generic fashion.

Method 1: The delete keyword. This will eliminate the value but not reindex the array nor modify its .length value. Take note of the terminal comma in the after case. The index persists even though the value does not. This can risk conceptual errors on the part of the web developer and is therefore not recommended within good reasoning. (Had the deleted element appeared mid-array, one would have seen consecutive commas to indicated the now-undefined value.) 

rainbow = [ 'red', 'orange', 'yellow', 'green', 'blue', 
           'indigo', 'violet', 'pink' ];

alert( 'before:\n' + rainbow 
            + '\nlength: ' + rainbow.length );

delete rainbow[ rainbow.lastIndexOf('pink') ];

alert( 'after:\n' + rainbow 
           + '\nlength: ' + rainbow.length );

Method 2: Use the very same .splice( ) method but include zero items after the set-up values. That is, N will be 0 and it will be the second and final argument passed to the method. 

rainbow = [ 'red', 'orange', 'yellow', 'green', 'blue', 
           'indigo', 'violet', 'pink' ];

alert( 'before:\n' + rainbow 
            + '\nlength: ' + rainbow.length );

rainbow.splice( rainbow.lastIndexOf('pink'), 1 );

alert( 'after:\n' + rainbow 
           + '\nlength: ' + rainbow.length );

It is perhaps best practice to master the .splice( ) method before worrying too much about all the other manipulations until a time when those options affect runtime in a professional web development setting.

A Few Cautions About Arrays

  1. So far, we have only seen arrays of strings. Because one could have an arrays of integers and because non-negative integers are also used as indices, confuse can arise. For example: consider having the array [10,11,12,13,14,15,16] and then using .splice( ) to change it to [10,11,4,5,6,7,8,15,16]. 

    numbers = [ 10,11,12,13,14,15,16 ];

alert( 'before:\n' + numbers 
            + '\nlength: ' + numbers.length );

numbers.splice( 2,3,4,5,6,7,8 );

alert( 'after:\n' + numbers 
           + '\nlength: ' + numbers.length );

    The 2 is an array position, the 3 is the quantity of elements to be removed, and the 4, 5, 6, 7, and 8 are all new elements inserted. However, in the .splice( ), it looks as if there will be a full sequence from 2 through 8.

  2. Legacy JavaScript/ECMA has an array constructor (new Array( )). When the constructor is given two or more values, it simply builds an array with those elements in order. However, when the constructor is given a single positive integer, it creates an array with that length and every element is undefined; It does not create an array consisting of only one element whose value is that integer. 

    one = new Array(5,10);
two = new Array(5);

alert( 'new Array(5,10): ' + one + ' has length: ' + one.length 
   + '\nnew Array(5): ' + two + ' has length: ' + two.length );

    Technically it's a feature; But it acts like a bug. Do not use the Array( ) constructor.

  3. Finally, as a separate concern, the empty string will be indistinguishable from a deleted element when a whole array is outputted in totality as a string.

Nested Arrays

Arrays are very versatile in that each and every element of the array can itself be of its own type of information. That is, one can mix and match strings, integers, floats, Booleans, Objects, and so on. This means that arrays can have other arrays as elements. 

// this is an empty object
three = new Object(); 

// this is an object with four defined fields
six = { year:2020, model:'sedan', price:21589, 
        color:'blue', list:[10,20,30] }; 

strangeArray = [
     0.123456789,
     '1',
     2,
     three, 
     [4.05, 4.15, 4.25, 4.35, 4.45, 4.55],
     ['five A', 'five B', 'five C', 'five D', 'five E'],
     six
  ]; // end of strangeArray definition

delete strangeArray[3];

alert( 'strangeArray: ' + strangeArray
      + '\nstrangeArray[0]: ' +  strangeArray[0]
      + '\nstrangeArray[1]: ' +  strangeArray[1]
      + '\nstrangeArray[2]: ' +  strangeArray[2]
      + '\nstrangeArray[3]: ' +  strangeArray[3]
      + '\nstrangeArray[4]: ' +  strangeArray[4]
      + '\nstrangeArray[5]: ' +  strangeArray[5]
      + '\nstrangeArray[6]: ' +  strangeArray[6]
  ); // end alert

alert( '\nstrangeArray[4][0]: ' +  strangeArray[4][0]
     + '\nstrangeArray[4][1]: ' +  strangeArray[4][1]
     + '\nstrangeArray[4][2]: ' +  strangeArray[4][2]
     + '\nstrangeArray[4][3]: ' +  strangeArray[4][3]
     + '\nstrangeArray[4][4]: ' +  strangeArray[4][4]
     + '\nstrangeArray[4][5]: ' +  strangeArray[4][5]
     + '\nstrangeArray[4][6]: ' +  strangeArray[4][6]
  ); // end alert 

alert( '\nstrangeArray[6].year: ' +  strangeArray[6].year 
     + '\nstrangeArray[6].model: ' +  strangeArray[6].model
     + '\nstrangeArray[6].price: ' +  strangeArray[6].price
     + '\nstrangeArray[6].color: ' +  strangeArray[6].color
     + '\nstrangeArray[6].list: ' +  strangeArray[6].list
     + '\nstrangeArray[6].list[2]: ' +  strangeArray[6].list[2]
     + '\nstrangeArray[6][0]: ' +  strangeArray[6][0]
  ); // end alert

In this example, the strangeArray is built out of a variety of data types. Note that each element needs to exist at the time of array assignment. It is perfectly fine that strangeArray[3] was deleted after-the-fact. Note that when the entire array is alerted, the nuances of multidimensional arrays gets lost. Also note that the object, which does have its own fields of information, one of which was itself an array, is only shown as a generic [object Object] when referenced as a whole.

Note that to access elements that are in arrays inside of the strangeArray will have multiple sets of square-brackets. In a similar vein, fields of objects cannot be accessed this way. Attempts to access elements that do not have corresponding indexes will be listed as undefined. The same is true of non-existent fields of objects. Pay attention to accessing an element within an array inside of an object inside of an array: (That was the strangeArray[6].list[2] example. Read this as The element at index 2 of the list field of the element at index 6 of the strange array.)
